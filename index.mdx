nuqs for SolidJS brings type-safe URL search parameter management to [SolidJS](https://solidjs.com). It lets you treat URL query strings as reactive signals — reading and writing them like any other SolidJS state, with full type safety and built-in parsers for common data types.

## Why nuqs for SolidJS?

URL query strings are the natural home for shareable, bookmarkable UI state — search terms, filters, pagination, and tabs. But parsing them safely and keeping them in sync with your reactive UI is tedious. nuqs for SolidJS handles that for you.

<CardGroup cols={2}>
  <Card title="Reactive signals" icon="bolt">

## How it works

<Steps>
  <Step title="Wrap your app with NuqsAdapter">
    Add the context provider once at your app root. It sets up URL observation and cross-component sync.

    ```tsx src/index.tsx
    import { render } from "solid-js/web";
    import { NuqsAdapter } from "./nuqs";
    import App from "./App";

    render(
      () => (
        <NuqsAdapter>
          <App />
        </NuqsAdapter>
      ),
      document.getElementById("root")!,
    );
    ```
  </Step>
  <Step title="Declare URL-backed state">
    Use `createQueryState` in any component — it reads from and writes to the URL automatically.

    ```tsx
    import { createQueryState, parseAsString, parseAsInteger } from "./nuqs";

    function SearchPage() {
      const [query, setQuery] = createQueryState("q", parseAsString);
      const [page, setPage] = createQueryState("page", parseAsInteger.withDefault(1));

      return (
        <div>
          <input
            value={query() ?? ""}
            onInput={(e) => setQuery(e.currentTarget.value || null)}
          />
          <p>Page {page()}</p>
        </div>
      );
    }
    ```
  </Step>
  <Step title="Sync multiple params together">
    Use `createQueryStates` to batch multiple URL parameters into a single reactive object — updates are batched into one history entry.

    ```tsx
    const [filters, setFilters] = createQueryStates({
      search: parseAsString.withDefault(""),
      page: parseAsInteger.withDefault(1),
    }, { history: "replace" });
    ```
  </Step>
</Steps>

<Tip>
  All URL updates are throttled to 50 ms by default to respect browser History API rate limits. See [Rate Limiting](/guides/rate-limiting) to customize this behavior.
</Tip>
