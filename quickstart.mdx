<Info>
This quickstart assumes you already have a SolidJS + Vite + TypeScript project. If you are starting from scratch, see [Installation](/installation) first.
</Info>

<Steps>

<Step title="Add the nuqs source to your project">

Because the library is not yet published to npm, copy the `src/nuqs/` directory from the repository into your project. Place it wherever you keep shared utilities — `src/nuqs/` is the convention used in this guide.

```
your-project/
└── src/
    ├── nuqs/          ← copied from the repo
    │   ├── index.ts
    │   ├── defs.ts
    │   ├── parsers.ts
    │   ├── useQueryState.ts
    │   ├── useQueryStates.ts
    │   └── adapters/
    │       └── solid.tsx
    └── App.tsx
```

<Tip>
Keep `src/nuqs/` as a self-contained directory. All public exports are re-exported through `src/nuqs/index.ts`, so you only ever import from `"./nuqs"` or `"../nuqs"`.
</Tip>

</Step>

<Step title="Wrap your app with NuqsAdapter">

`NuqsAdapter` is a context provider that owns the URL state. It must be an ancestor of every component that calls `createQueryState` or `createQueryStates`. The right place is your app entry point — typically `src/index.tsx`.

```tsx src/index.tsx
/* @refresh reload */
import { render } from "solid-js/web";
import App from "./App.tsx";
import { NuqsAdapter } from "./nuqs";

const root = document.getElementById("root");

render(
  () => (
    <NuqsAdapter>
      <App />
    </NuqsAdapter>
  ),
  root!,
);
```

`NuqsAdapter` accepts two optional props:

| Prop | Type | Description |
|---|---|---|
| `defaultOptions` | `Partial<Options>` | Default `scroll`, `shallow`, `limitUrlUpdates`, and `clearOnDefault` for all hooks in the tree. Note: `history` and `throttleMs` are not configurable at the adapter level. |
| `processUrlSearchParams` | `(search: URLSearchParams) => URLSearchParams` | Transform search params before they are written to the URL — useful for preserving params your app does not manage. |

</Step>

<Step title="Manage a single URL parameter">

`createQueryState(key, parser)` returns a `[accessor, setter]` pair — the same shape as SolidJS's `createSignal`. Call the accessor in reactive contexts (JSX, `createEffect`, `createMemo`); call the setter to update the URL.

```tsx src/App.tsx
import { createQueryState, parseAsString, parseAsInteger } from "./nuqs";

function App() {
  // "name" is string | null — no default
  const [name, setName] = createQueryState("name", parseAsString);

  // "count" is number — defaults to 0 when the key is absent from the URL
  const [count, setCount] = createQueryState(
    "count",
    parseAsInteger.withDefault(0),
  );

  return (
    <>
      <section>
        <h2>Name</h2>
        <input
          type="text"
          value={name() ?? ""}
          onInput={(e) => setName(e.currentTarget.value || null)}
          placeholder="Your name"
        />
        <button onClick={() => setName(null)}>Clear</button>
        <p>Hello, {name() ?? "anonymous visitor"}!</p>
      </section>

      <section>
        <h2>Counter</h2>
        <p>Count: {count()}</p>
        <button onClick={() => setCount((c) => (c ?? 0) + 1)}>+</button>
        <button onClick={() => setCount((c) => (c ?? 0) - 1)}>-</button>
        <button onClick={() => setCount(0)}>Reset</button>
        <button onClick={() => setCount(null)}>Remove from URL</button>
      </section>
    </>
  );
}
```

**What the setter accepts:**

- A new value: `setName("Alice")`
- `null` to remove the key from the URL: `setName(null)`
- An updater function that receives the current value: `setCount((prev) => (prev ?? 0) + 1)`
- An optional second argument `Options` to override history mode, scroll, etc. per call: `setName("Alice", { history: "push" })`

<Note>
When a value equals its `.withDefault(value)` value, the key is removed from the URL automatically (`clearOnDefault: true` by default). This keeps URLs clean — `?count=0` does not appear when `0` is the default.
</Note>

</Step>

<Step title="Manage multiple URL parameters at once">

`createQueryStates(keyMap, options)` lets you declare and update several parameters as a group. Updates to multiple keys are applied in a single `history` call, which prevents intermediate states from appearing in the URL.

```tsx src/App.tsx
import {
  createQueryState,
  createQueryStates,
  parseAsString,
  parseAsInteger,
} from "./nuqs";

function App() {
  const [name, setName] = createQueryState("name", parseAsString);
  const [count, setCount] = createQueryState(
    "count",
    parseAsInteger.withDefault(0),
  );

  // Declare a group of related params
  const [filters, setFilters] = createQueryStates(
    {
      search: parseAsString.withDefault(""),
      page: parseAsInteger.withDefault(0),
    },
    { history: "replace" },
  );

  return (
    <>
      {/* Single-key examples from Step 3 */}

      <section>
        <h2>Filters</h2>
        <input
          type="text"
          value={(filters().search as string) ?? ""}
          onInput={(e) => setFilters({ search: e.currentTarget.value })}
          placeholder="Search..."
        />
        <p>Page: {filters().page as number}</p>
        <button
          onClick={() =>
            setFilters({ page: ((filters().page as number) ?? 0) + 1 })
          }
        >
          Next page
        </button>
        <button
          onClick={() =>
            setFilters({
              page: Math.max(0, ((filters().page as number) ?? 0) - 1),
            })
          }
        >
          Prev page
        </button>
      </section>
    </>
  );
}
```

`filters()` returns an object whose keys match the `keyMap` you passed in. You can update any subset of keys by passing a partial object to `setFilters`. Keys you omit are left unchanged.

**Updater function:**

```tsx
// Update page relative to current state
setFilters((prev) => ({ page: prev.page + 1 }));
```

</Step>

</Steps>

## What you now have

After completing these steps, your application URL reflects every state change. Open the browser address bar and you will see params like `?name=Alice&count=3&search=solid&page=1`. Sharing that URL restores the exact state.
